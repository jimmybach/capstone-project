Gradient Boosting in Localized Weighted Regression

Introduction

Gradient Boosting is a machine learning technique in which which an originally weak learner becomes stronger through multiple iterations. Whereas other popular boosting techniques such as Adaboost tweak the original model's weights that were fit on the training data, gradient boosting builds an entirely new model that's fit with the target training values instead being replaced by the residuals of the model. This process is repeated further, with each subsequent model being trained on the residuals of the previous model. We can control the learning rate, or eta, which is a hyperparameter that we can multiply the residuals by to determine how quickly we wish to try and descend. Since we're looking to minimize our cost function, we want to find the correct eta that avoids overshooting or undershooting this minimum in the number of times we boost our model. A model that is boosted 3 times would look like the following:

(1)y^=y^1+ηr1+ηr2

Where y1 is the training predictions and r1 and r2 are the residuals of the first and second models.

Applying Gradient Boosting with Localized Weighted Regression

Like many other forms of machine learning, gradient boosting can greatly improve the performance of localized weighted regression. The following examples showcase this with the oft-used Concrete Compressive Strength dataset. First, for full transparency and reader understanding, I present my localized weighted regression class below, which includes a gradient boosting function in the form of the gradient_boost method. For further details on the workings of the code as well as more background information on the theory of localized weighted regression, I encourage you to read my prior post for a more thorough breakdown of what is actually happening (the grid_search method is omitted since it is not used in this example).

Loess Class

class loess:


    def __init__(self, kernel=Gaussian, tau=0.5,scaler=SS):
          self.kernel = kernel
          self.tau = tau
          self.fitted=False
          self.scaler=scaler()


    def fit(self, x, y):
          self.xtrain_=self.scaler.fit_transform(x)
          self.yhat_ = y.reshape(-1,1)
          self.fitted=True
    
    def is_fitted(self):
      if self.fitted==False:
        raise "Please fit your model first"


    def predict(self,x_new,model=Ridge(alpha=0.01)):
            self.is_fitted()
            self.xtest_=self.scaler.transform(x_new)
            w = self.kernel(cdist(self.xtrain_, self.xtest_, metric='euclidean')/(2*self.tau))


            local_model=model


            if np.isscalar(self.xtest_):
              local_model.fit(np.diag(w)@(self.xtrain_.reshape(-1,1)),np.diag(w)@(self.yhat_.reshape(-1,1)))
              yest_test = local_model.predict([[self.xtrain_]])[0][0]
            else:
              n = len(self.xtest_)
              yest_test = np.zeros(n)
              for i in range(n):
                local_model.fit(np.diag(w[:,i])@(self.xtrain_),np.diag(w[:,i])@(self.yhat_.reshape(-1,1)))
                yest_test[i] = local_model.predict(self.xtest_[i].reshape(1,-1))
                
            return yest_test




    def gradient_boost(self,y_test=None,is_predicted=False,num_iter=10,eta=1,local_model=Ridge(alpha=0.01)):
      self.is_fitted()


      m1=loess(self.kernel,self.tau)
      m1.fit(x=self.xtrain_,y=self.yhat_.flatten())
      m1_predictions=m1.predict(self.xtrain_)


      residuals=self.yhat_.flatten()-m1_predictions
      m2=loess(self.kernel,self.tau)
      m2.fit(self.xtrain_,residuals)
      m2_predictions=m2.predict(self.xtrain_)
      test_predictions=m1.predict(self.xtest_)
      mean_squared_errors=[mse(test_predictions,y_test)]
      sign=1
      for i in range(num_iter):
        residuals=residuals-eta*m2_predictions
        m2.fit(self.xtrain_,residuals)
        m2_predictions=m2.predict(self.xtrain_)
        test_predictions+=sign*m2.predict(self.xtest_)
        mean_squared_errors.append(mse(test_predictions,y_test))
        if mean_squared_errors[-1]>mean_squared_errors[-2]:
          mean_squared_errors.append(mse(test_predictions,y_test))
          sign*=-1
      return np.array(mean_squared_errors)




    def do_Kfold(self, X, y, model=Ridge(alpha=0.01),boosted=False,k=10, random_state = 146):


      kf = KFold(n_splits=k, random_state = random_state, shuffle=True)


      test_scores = []


      for idxTrain, idxTest in kf.split(X):
          Xtrain = X[idxTrain, :]
          Xtest = X[idxTest, :]
          ytrain = y[idxTrain]
          ytest = y[idxTest]


          self.fit(Xtrain,ytrain)
          predictions=self.predict(Xtest,model=model)
          if boosted:
            test_scores.append(self.gradient_boost(ytest,num_iter=5)[-1])
          else:
            test_scores.append(mse(predictions,ytest))


      return np.mean(test_scores)

The gradient_boost function has built-in functionality to avoid overshooting the minimum of the cost. It takes the MSE of new predictions and the actual values for every iteration ran. If the last recorded MSE is larger than the second to last one, the sign of the residual calculation flips (i.e. we would subtract instead of add, or add instead of subtract if it had already flipped before). The method returns the MSE for every boost of the model.

Comparing Different Scalers with 10-fold Cross-Validation and Gradient Boosting

To understand why we even need to scale the data in the first place, I present a strip plot of the scale for each feature.

The scales that each feature operates on are wildly different. While we could choose to use a local model for our regression that is independent of scaling, The local weights assigned to each point are dependent on the real values of the features. Thus, if one variable operates on a drastically larger scale than another, it can hold more influence over the weights.

Localized Weighted Logistic Regression

Introduction

Loess models can also be used for classification, not just regression. One of the most common classification machine learning techniques is logistic regression. Where for regression techniques the cost function that is being minimized is often mean squared error, logistic regression looks to minimize the following function (in the case of 2 categories):

(2)loss=−y∗ln(p)−(1−y)∗ln(1−p)

Where (p,1-p) are the predicted probabilities for classes (y,1-y), which will either have a value of 0 or 1. This function is better known as binary cross-entropy. To get the probabilities, logistic regression utilizes the sigmoid function, shown below:

(3)p(x)=11+e−(β0+β1x)

Application of Localized Weighted Logistic Regression

I present two classes: my own (loess_classification) and Calvin Chi's (locally_weighted_logistic_regression), found here. Whereas mine simply utilizes Sci-Kit Learn's LogisticRegression class, Calvin's operates to update the weights of the linear combinations for each test point using the Newton-Raphson algorithm, which subtracts the weights by a Hessian matrix multiplied by the gradient of a maximum likelihood loss function. From there, I compare the success of the two different classes by performing binary logistic regression on the iris dataset 3 times and checking the accuracy of the respective predictions using the accuracy_score method.

class loess_classification:


    def __init__(self, kernel=Gaussian, tau=0.5):
          self.kernel = kernel
          self.tau = tau
          self.fitted=False


    def fit(self, x, y):
          self.ss=SS()
          self.xtrain_=self.ss.fit_transform(x)
          self.yhat_ = y
          self.fitted=True


    def is_fitted(self):
      if self.fitted==False:
        raise "Please fit your model first."


    def predict(self,x_new,model=LR()):
            self.is_fitted()
            self.xtest_=self.ss.transform(x_new)
            w = self.kernel(cdist(self.xtrain_, self.xtest_, metric='euclidean')/(2*self.tau))
            local_model=model


            if np.isscalar(self.xtest_):
              local_model.fit(np.diag(w)@(self.xtrain_.reshape(-1,1)),(self.yhat_))
              yest_test = local_model.predict([[self.xtrain_]])[0][0]
            else:
              n = len(self.xtest_)
              yest_test = np.zeros((n,len(self.yhat_[0])))
              for i in range(n):
                local_model.fit(np.diag(w[:,i])@(self.xtrain_),(self.yhat_))
                yest_test[i] = local_model.predict(self.xtest_[i].reshape(1,-1))
            return yest_test


class locally_weighted_logistic_regression(object):


    def __init__(self, tau, reg = 0.0001, threshold = 1e-6):
        self.reg = reg
        self.threshold = threshold
        self.tau = tau
        self.w = None
        self.theta = None
        self.x = None


    def weights(self, x_train, x):
        sq_diff = (x_train - x)**2
        norm_sq = sq_diff.sum(axis = 1)
        return np.ravel(np.exp(- norm_sq / (2 * self.tau**2)))


    def logistic(self, x_train):
        return np.ravel(1 / (1 + np.exp(-x_train.dot(self.theta))))


    def train(self, x_train, y_train, x):
        self.w = self.weights(x_train, x)
        self.theta = np.zeros(x_train.shape[1])
        self.x = x
        gradient = np.ones(x_train.shape[1]) * np.inf
        while np.linalg.norm(gradient) > self.threshold:
            # compute gradient
            h = self.logistic(x_train)
            gradient = x_train.T.dot(self.w * (np.ravel(y_train) - h)) - self.reg * self.theta
            # Compute Hessian
            D = np.diag(-(self.w * h * (1 - h)))
            H = x_train.T.dot(D).dot(x_train) - self.reg * np.identity(x_train.shape[1])
            # weight update
            self.theta = self.theta - np.linalg.inv(H).dot(gradient)


    def predict(self):
        return np.array(self.logistic(self.x) > 0.5).astype(int)

Within the confusion matrices that I present, the correct cases fall upon the main diagonal of the matrix (the top left and bottom right squares), and the incorrect cases fall outside this main diagonal (the top right and bottom left squares).
