Classification with Class Imbalance: An Application of Different Methods

 

There are many problems in the world of data science where we have imbalanced classes. A classification dataset is considered to be imbalanced if there are more observations with one class for the target value than there are another. In the case of the dataset analyzed in this application, the Porto Seguro Safe Driver dataset, the data is strongly imbalanced, with over 95% of drivers in the dataset not having a claim filed. I'll be looking at 3 different techniques which attempt to combat the imbalanced dataset by artificially generating data. These three include:

  • Synthetic Minority Oversampling Technique (SMOTE)

  • Adaptive Synthetic Oversampling (ADASYN) with Fast Kernel Density Estimation (FastKDE)

  • Normalizing Flows

To compare the three, confusion matrices, recall scores, and accuracy scores will be presented. It's important to go beyond simply reporting accuracy in the case of heavily imbalanced datasets. When modeling imbalanced data without the help of some of the methods shown (and other similar ones), the model tends to present high accuracy, as it only predicts the majority class, which is heavily present in any validation data used. However, there is also very low recall in these situations, as the predictive power of the minority class is near-nonexistent. Thus, having both of these metrics available can show how these techniques affect this problem.

SMOTE

SMOTE involves synthetically oversampling (adding more observations) to the minority class by developing fictitious data based off of what already exists. It develops datapoints based off of the nearest neighbors to a point that is in the minority class and calculating synthetic observations in between the point and the neighbors. Here's how SMOTE performs as the training data for a basic Logistic Regression model, with actual data serving as the test data. This pipeline of oversampling and modeling was all cross-validated using StratifiedKFolds, and the mean accuracy scores, recalls, and overall confusion matrices are presented.

skf = StratifiedKFold(n_splits=10,random_state=440,shuffle=True)
lr=LR(random_state=440)
imba_pipeline = make_pipeline(SMOTE(random_state=440), 
                              lr)
recalls=np.mean(cross_val_score(imba_pipeline, X, y, scoring='recall', cv=skf))
accuracies=np.mean(cross_val_score(imba_pipeline, X, y, scoring='accuracy', cv=skf))
y_pred = cross_val_predict(imba_pipeline, X, y, cv=skf)
conf_mat = cm(y, y_pred)
disp=cmd(conf_mat,display_labels=['No Claim','Claim'])
print(f'Stratified Cross-Validated SMOTE Recall: {recalls:.4f}')

ADASYN/FastKDE

FastKDE is a Kernel Density Estimation method which is used to quickly estimate a probability density function with finite amounts of data. It uses adaptive bandwidth selection to form local relationships within different points in the distribution. The "Fast" component comes from the method's usage of the Fast Fourier Transform to reduce overall computational complexity. This complexity reduction leads to speedier calculations. Overall, it provides a quicker, more locally accurate, flexible, and usable form of PDF estimation than previous methods, making it useful for a dataset of this size and complexity.

ADASYN aims to accomplish the same goals as SMOTE (oversampling the minority class), but goes about doing so in a different way. Whereas SMOTE was interpolating between minority class observations to make new points, ADASYN adaptively creates new samples via the density distribution of the minority observations. With this in mind, it's clear how FastKDE and ADASYN can be used as complementary methods to each other. 

xxxxxxxxxx
# Generate samples from the estimated distribution
# Get the density and axes
density = kde.pdf
axes = kde.axes
X1 = axes[0]  # Grid points along x-axis
X2 = axes[1] # Grid points along y-axis


# Create meshgrid for plotting
X1_grid, X2_grid = np.meshgrid(X1, X2)


# Flatten the density and grids
density_flat = density.flatten()
X1_flat = X1_grid.flatten()
X2_flat = X2_grid.flatten()


# Normalize the density to create a probability mass function (PMF)
pmf = density_flat / density_flat.sum()


# Create the cumulative distribution function (CDF)
cdf = np.cumsum(pmf)


# Number of samples to generate
n_samples_to_generate = y.shape[0]


# Generate uniform random samples between 0 and 1
random_values = np.random.rand(n_samples_to_generate)


# Map the random values to indices in the CDF
indices = np.searchsorted(cdf, random_values)


# Get the corresponding samples from the grid
samples_x1 = X1_flat[indices]
samples_x2 = X2_flat[indices]


# Stack the samples into an array of shape (n_samples, 2)
samples = np.vstack((samples_x1, samples_x2)).T


X_kde=pd.DataFrame(samples)


lr=LR(random_state=440)
imba_pipeline = make_pipeline(ADASYN(random_state=440), 
                              lr)
recalls=np.mean(cross_val_score(imba_pipeline, X_kde, y, scoring='recall', cv=skf))
accuracies=np.mean(cross_val_score(imba_pipeline, X_kde, y, scoring='accuracy', cv=skf))
y_pred = cross_val_predict(imba_pipeline, X, y, cv=skf)
conf_mat = cm(y, y_pred)
disp=cmd(conf_mat,display_labels=['No Claim','Claim'])
print(f'Stratified Cross-Validated ADASYN/FastKDE Recall: {recalls:.4f}')

Normalizing Flows

Normalizing Flows helps convert a simple probability distribution into a more complex one via mappings which are both invertible and differentiable. One major benefit of Normalizing flows is the ability of the method to compute exact likelihoods using the log-density of the function compositions, which leads to simpler training processes. The tradeoff of this is that it is a more computationally complex algorithm that doesn't translate very well to higher dimensional space. It can support various types of transformations within the function compositions, including autoregressive flows, affine coupling, and spline flows.

xxxxxxxxxx
num_iter = 5000
for i in range(num_iter):
    flows_train=np.random.choice(X.shape[0],1000,replace=False)
    X_flows=X[flows_train]
    y_flows=y[flows_train]
    
    X_flows = torch.tensor(np.array(X_flows), dtype=torch.float32, device=device)
    y_flows = torch.tensor(np.array(y_flows), dtype=torch.float32, device=device).reshape(-1, 1)
    
    optimizer.zero_grad()
    loss = -flow.log_prob(inputs=X_flows, context=y_flows).mean()
    loss.backward()
    optimizer.step()


recalls=[]
accuracies=[]
all_y=[]
all_preds=[]
for i, (train_index, test_index) in enumerate(skf.split(X, y)):
    X_test,y_test=X[test_index],y[test_index]




    # Lets create two hypothesis and assume we do not know the ground truth labels
    # Hypothesis of class 0
    x_ = torch.tensor(X_test,device=device,dtype=torch.float32)
    hyp_0 = torch.tensor(np.zeros_like(y_test),device=device,dtype=torch.float32).reshape(-1,1)
    # Hypothesis of class 1
    hyp_1 = torch.tensor(np.ones_like(y_test),device=device,dtype=torch.float32).reshape(-1,1)


    LL_class_0 = flow.log_prob(x_,context=hyp_0).detach().cpu().numpy()
    LL_class_1 = flow.log_prob(x_,context=hyp_1).detach().cpu().numpy()
    # We can make predictions based off which likelihood is larger
    y_pred = np.zeros_like(y_test)


    y_pred[LL_class_1 > LL_class_0] = 1.0


    recalls.append(recall_score(y[test_index],y_pred))
    accuracies.append(accuracy_score(y[test_index],y_pred))
    all_y.append(y[test_index])
    all_preds.append(y_pred)
recalls=np.mean(recalls)
accuracies=np.mean(accuracies)
conf_mat = cm(np.concatenate(all_y), np.concatenate(all_preds))

Comparing Results

Normalizing Flows did the best job of classifying the minority class and didn't sacrifice very much in the majority class either. While it had the lowest accuracy at about 59%, this wasn't demonstratively lower than the other two methods and the recall of the minority classes was far more successful, as it correctly classified over 1200 of the minority observations. The opposite was true for ADASYN and FastKDE, as there was a drastic dropoff in recall but the accuracy was highest overall, showing that the PDF from FastKDE may have been too overfit to the majority class. SMOTE was somewhere in the middle, with a recall of about 50% and an accuracy of about 60%.
